# Revertendo commits com revert

O revert é um "ctrl z" para os seus commits, ele cria um commit novo mostrando qual commit foi revertido, assim não estragando o histórico do git log.
Comando:
> git revert hashdocommit

Esse comando é mais indicado para se usar no master para manter o histórico.

**Boa prática:**
- NUNCA ALTERE O HISTÓRICO NO MASTER;
- Se usa o revert para o master e o reset para as outras branchs.
