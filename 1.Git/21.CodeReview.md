# Code review e squash no merge

A parte de code review no github é bem intuitiva, os responsáveis por revisar o código darão seus pitacos e o projeto vai seguir.

Depois de finalizado o code review, se for o caso dos commits que subiram serem meras correções bobas é possível usar o **"squash and merge"** no github para deixar o histórico de commit mais limpo.

Caso a feature que esteja subindo para o repositório remoto seja grande, ai realmente pode ser mais aconselhável fazer o squash no terminal e o merge pull request direto.
