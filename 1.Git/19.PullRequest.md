# O que é o pull request?

Quando você cria uma branch, faz suas alterações e sobe para o repositório remoto, é interessante que outros desenvolvedores revisem o código para corrigir erros antes de mandar para a branch principal, é para isso que serve o pull request.

Ao subir as modificações no github ele vai notificar que tem mudanças no código e vai segurir o pull request para ser feito.

## Template pull request

Não é muito diferente dos templates das issues, basta criar um arquivo no **.github** denominado **pull_request_template.md**.
