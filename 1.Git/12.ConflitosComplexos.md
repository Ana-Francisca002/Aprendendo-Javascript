# Conflitos mais complexos

Imagine a seguinte situação:
Você fez alterações em determinadas partes do código e fez o commit. Outra pessoa, sem saber, acabou modificando as mesmas partes que você e subiu para o repositório remoto, você tentou dar o git pull e o conflito se instaurou no código.

Para solucionar o git vai criar linhas extras mostrando as duas modificações, você vai ter que adicionar as mudanças de cada arquivo:
> git add nomedoarquivo

Com isso, basta fazer o merge continuar:
> git merge --continue 
Esse parâmetro do continue é justamente para essas situações de conflito, se você fizer o git merge sem o parâmetro terá mais problemas.
