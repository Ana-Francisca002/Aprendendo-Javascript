# O que é DNS?

Como já vimos anteriormente, o DNS serve para "traduzir" os Nomes de domínio em endereços IP.

## Como funciona?

Vamos entender as etapas que ocorrem no DNS para que ele forneça as informações necessárias para cada dispositivo.

- **Recursor de DNS:** O recursor de DNS é um servidor projetado para receber consultas de máquinas clientes por meio de aplicativos como navegadores web;
- **Servidor raíz:** A primeira etapa da tradução (resolução) de host names legíveis por humanos em endereços de IP;
- **Nameserver TDL:** Esse nameserver é o próximo passo na busca de um endereço de IP específico e hospeda a última parte de um hostname (.com);
- **Nameserver autoritativo:** A última parada na consulta de um servidor de DNS. Se tiver acesso ao registro solicitado, o nameserver autoritativo retornará o endereço de IP do hostname solicitado de volta ao recursor de DNS (o bibliotecário) que fez a solicitação inicial.

## Três tipos de consultas de DNS:

- **Consulta recursiva:** Nesta consulta, o cliente de DNS pede que o servidor responda com o registro do recurso desejado ou um erro, se não encontrado.;

- **Consulta iterativa:** Aqui, o cliente permite que o servidor retorne a melhor resposta possível. Se não houver correspondência, o servidor indica ao cliente qual servidor consultar a seguir;

- **Consulta não recursiva:** Esse tipo ocorre quando o resolvedor de DNS consulta um servidor autoritativo ou usa registros em cache para obter uma resposta sem precisar fazer novas consultas.
