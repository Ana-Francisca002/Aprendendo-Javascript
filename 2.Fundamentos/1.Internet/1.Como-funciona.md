# Como a internet funciona?

## Introdução 

Primeiramente, vamos entender o que é uma rede.

Uma **rede** são vários dispositios conectados um ao outro, sejam smartphones, computadores, etc. A internet surgiu quando foi possível unir essas várias redes de dispositivos com outras várias redes de dispositivos, até milhares de computadores estarem conectados no mundo todo.

> A internet é a rede das redes.

Um pouco sobre a história da internet, bom, ela foi desenvolvida no final de 1960 pelo Depertamento de Defesa dos Estdados Unidos como meio de comunicação para períodos de guerra.

## Visão geral

Nessa grande rede de dispositivos conectados e usando um conjunto de **protocolos padronizados** que a troca de informações acontece, sendo de responsabilidade dos protocolos garantir a confiabilidade e segurança dos dados.

A internet trasmite esses dados quebrando os pacotes em unidades menores para que sejam examinados pelo roteador, para que assim o pacote siga para o seu destino final.



## Conceitos báscios

- **Pacote:** A menor unidade de dados;
- **Roteador:** Dispositivo responsável por direcionar os pacotes na rede;
- **Endereço de IP:** Identificador único atribuído a cada dispositivo, isso garante que os dados vão chegar no destino correto;
- **Nome de domínio:** Os endereços de IP legíveis para humanos (EX: google.com | IP: 8.8.8.8)
- **DNS:** Traduz os domínios em endereços ip;
- **HTTP:** Usado para transferir dados entre o cliente (navegador) e server (site);
- **HTTPS:** Versão criptografada do HTTP;
- **SSL/TLS:** Responsável por fornecer uma camada de segurança a mais nas comunicações na internet (é a camada de segurança do http).



## Construindo aplicações com TCP/IP

O TCP/IP é o protocolo que garante a confiabilidade, ordenamento e entrega com verificação de erros dos dados com aplicações rodando em diferentes dispositivos.

Conceitos fundamentais:
- **Porta:** É o identificador dos serviços e aplicativos, por exemplo: 25 (e-mail), 80 (tráfego web), 443 (tráfego web criptografado). É padronizado para todos os dispositivos conectados a internet;
- **Soquete:** A combinação entre o endereço IP e as portas, estabelecendo um ponto final no fluxo de comunicação;
- **Conecxões:** A conexão é quando os dois soquetes estabilizam, permitindo a comunicação dos dispositivos;
- **Transferência de dados:** Quando a conexão é estabelecida, os dados podem ser transferidos entre os aplicativos em execução e os dispositivos.

### Aprofundando conceitos do TCP/IP

Para reforçar, o TCP é responsável por garantir a transmissão dos dados e o IP endereça os dispositivos na rede.
O modelo TCP/IP, com seu conjunto de protocolos, define como os dados são divididos em pacotes para trafegarem na rede.

Esse protocolos são agrupados em camadas, são elas:

- **Aplicação:** Onde os dados são preparados para serem enviados pela rede;
- **Trasporte:** Os dados são divididos em pacotes para serem enviados
- **Internet/rede:** O IP atua principalmente nessa camada, endereçando os pacotes;
- **Acesso à rede/enlance:** Essa é a que faz a transmissão física dos dados.

![Camadas TCP/IP](/2.Fundamentos/1.Internet/img/tcp-ip-1060x596.png)

Os protocolos que integram cada uma dessas camadas são:

- **Aplicação**
    - **HTTP (HyperText Transfer Protocol):** Usado para comunicação entre navegadores e servidores web, permitindo a transferência de páginas web e outros recursos;
    - **HTTPS (HyperText Transfer Protocol Secure):** Uma versão segura do protocolo HTTP;
    - **FTP (File Transfer Protocol):** Permite a transferência de arquivos entre computadores;
    - **SMTP (Simple Mail Transfer Protocol):** Usado para o envio e recebimento de dados na comunicação entre servidores de e-mail;
    - **POP3 (Post Office Protocol version 3) e IMAP (Internet Message Access Protocol):** Usados para receber e gerenciar e-mails em um servidor de correio;
    - **DHCP (Dynamic Host Configuration Protocol):** Atribui automaticamente endereços IP aos dispositivos conectados à rede, facilitando a configuração e a administração;
    - **DNS (Domain Name System):** Traduz os nomes de domínio, como “www.tecnoblog.net”, em endereços IP numéricos, tornando mais fácil para os usuários acessarem sites.

- **Transporte**
    - **TCP:** Quem garante a entrega confiável dos dados;
    - **UDP:** Permite o envio de dados com ampla velocidade, mas sem a verificação da entrega ou retransmissão de pacotes perdidos.

- **Rede**
    - **IP:** Endereça os pacotes.
